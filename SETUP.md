# New Machine Setup

After cloning the repo and building (`go build .`), run these steps to set up your exercism workspace for a smooth experience.

## Prerequisites

1. Install and configure the [Exercism CLI](https://exercism.org/docs/using/solving-exercises/working-locally):
   ```bash
   exercism configure --token=YOUR_TOKEN
   ```

2. Build the TUI:
   ```bash
   go build -o coding-tui .
   ```

## Fix ESLint Noise on Existing Exercises

ESLint flat config doesn't cascade — each exercise has its own `eslint.config.mjs`. The TUI patches these automatically on download, but exercises downloaded via the `exercism` CLI or before the TUI was installed need manual patching.

### Patch all JS/TS exercises

```bash
python3 << 'EOF'
import glob, os

marker = "// <<inject-rules-here>>"
patch = """// <<inject-rules-here>> (patched by coding-tui)
  {
    rules: {
      "no-unused-vars": "off",
      "@typescript-eslint/no-unused-vars": "off",
      "no-constant-condition": "off",
      "no-empty-function": "off",
      "no-unreachable": "off",
    },
  },"""

workspace = os.path.expanduser("~/.config/exercism/user.json")
try:
    import json
    with open(workspace) as f:
        root = json.load(f)["workspace"]
except Exception:
    root = os.path.expanduser("~/exercism")
    print(f"Could not read exercism config, using default: {root}")

count = 0
for pattern in [f"{root}/javascript/*/eslint.config.mjs",
                f"{root}/typescript/*/eslint.config.mjs"]:
    for path in glob.glob(pattern):
        with open(path) as f:
            content = f.read()
        if marker in content and "coding-tui" not in content:
            content = content.replace(marker, patch, 1)
            with open(path, "w") as f:
                f.write(content)
            exercise = os.path.basename(os.path.dirname(path))
            print(f"  patched: {exercise}")
            count += 1

print(f"\nDone. Patched {count} exercises.")
EOF
```

### Install shared node_modules

JS/TS exercises share dependencies. Install once at the track root:

```bash
# Read your workspace path
WORKSPACE=$(python3 -c "import json; print(json.load(open('$HOME/.config/exercism/user.json'))['workspace'])")

# JavaScript
if [ -d "$WORKSPACE/javascript" ]; then
  cd "$WORKSPACE/javascript"
  # Copy package.json from any exercise if not present at root
  if [ ! -f package.json ]; then
    cp $(ls */package.json | head -1) package.json
  fi
  npm install
  echo "✓ javascript node_modules installed"
fi

# TypeScript
if [ -d "$WORKSPACE/typescript" ]; then
  cd "$WORKSPACE/typescript"
  if [ ! -f package.json ]; then
    cp $(ls */package.json | head -1) package.json
  fi
  npm install
  echo "✓ typescript node_modules installed"
fi
```

### Ruby (rubocop)

```bash
WORKSPACE=$(python3 -c "import json; print(json.load(open('$HOME/.config/exercism/user.json'))['workspace'])")

cat > "$WORKSPACE/ruby/.rubocop.yml" << 'RUBOCOP'
# Auto-generated by coding-tui
AllCops:
  DisabledByDefault: true
Lint/UnusedMethodArgument:
  Enabled: false
Lint/UnusedBlockArgument:
  Enabled: false
Lint/UselessAssignment:
  Enabled: false
Style/FrozenStringLiteralComment:
  Enabled: false
Metrics:
  Enabled: false
RUBOCOP
echo "✓ ruby rubocop config written"
```

## Verify

Restart your editor after running these steps. ESLint warnings about unused variables should be gone.

Any new exercises downloaded through the TUI (`d` key) will be patched automatically.
